# Linux Packaging

UABEANext ships Linux packages as self-contained runtime builds. Users do not
need `dotnet` or a .NET runtime installed to run the generated package.

## Targets

- `linux-x64`: primary target for Arch Linux and Manjaro.
- `linux-musl-x64`: fallback target for systems where glibc compatibility is a
  startup blocker.

Use `linux-x64` first on Manjaro/Arch. It includes the Linux native texture
encoder libraries from `NativeLibs/linux-x64`. The musl package intentionally
does not include those glibc-only native texture encoder libraries, so
texture import/re-encode paths that need `textureencoder`, `cuttlefish`, or
`PVRTexLib` may be unavailable there.

## Build Packages

From the repository root:

```bash
eng/linux/package-linux.sh --rid linux-x64
eng/linux/package-linux.sh --rid linux-musl-x64
```

Artifacts are written to `artifacts/linux`:

```text
uabea-next-linux-x64.tar.gz
uabea-next-linux-musl-x64.tar.gz
```

The build machine needs the .NET SDK. The generated package does not.

## Manjaro/Arch Runtime Packages

```bash
sudo pacman -S --needed \
  fontconfig freetype2 libx11 libxext libxrandr libxrender libxcursor \
  libxi libxinerama libglvnd mesa gtk3 xdg-desktop-portal
```

## Manjaro/Arch Pacman Package

`uabea-next-linux-x64.tar.gz` is a plain portable tarball, not a pacman package.
Do not install it with `pacman -U`.

After building `uabea-next-linux-x64.tar.gz`, wrap it in a local pacman package:

```bash
eng/linux/package-arch.sh
sudo pacman -U artifacts/pkgbuild/uabea-next-bin-0.0.0-1-x86_64.pkg.tar.zst
```

The package installs the self-contained app under `/opt/uabea-next` and creates
`/usr/bin/uabea-next`. The package does not depend on dotnet.

## Debian/Ubuntu Runtime Packages

```bash
sudo apt install \
  libfontconfig1 libfreetype6 libx11-6 libxext6 libxrandr2 libxrender1 \
  libxcursor1 libxi6 libxinerama1 libgl1 libgtk-3-0 xdg-desktop-portal
```

## Running

Extract the tarball and run:

```bash
./uabea-next
```

The wrapper starts `app/UABEANext4.Desktop`, which is the native apphost
generated by `dotnet publish --self-contained true`.
